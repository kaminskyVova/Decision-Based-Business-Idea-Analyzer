Decision-Based Business Idea Analyzer
Система строгой оценки и допуска бизнес-идей.

Описание проекта

Decision-Based Business Idea Analyzer — это многослойная аналитическая система для строгой проверки бизнес-идей.
Проект предназначен для оценки целесообразности запуска инициатив на основе фактов, ограничений и ресурсов, а не интуиции или предположений.

Система не генерирует идеи и не принимает решения за пользователя.
Её задача — проверка возможности, выявление ограничений и формирование обоснованного аналитического вердикта.

Назначение системы

Проект решает ключевую проблему современных AI-ассистентов и агентных систем — отсутствие формального допуска к анализу.

В большинстве решений:

анализ запускается без достаточных данных;
система “догадывается” вместо проверки;
отсутствует ответственность за итоговое решение.
Данный проект вводит обязательный контур допуска (Gatekeeper) и запускает анализ только после прохождения всех проверок.

Архитектура (высокоуровневая)

Система состоит из трёх ключевых блоков:
Gatekeeper (Ворота)
Analysis (Анализ)
Plan (План действий)
Порядок выполнения фиксирован и не нарушается.
Идея → Gatekeeper → Analysis → Decision → Plan (опционально)


1. Gatekeeper (ядро системы)
Gatekeeper — это обязательный первый этап обработки любой идеи.

Задачи Gatekeeper:
определить, готова ли идея к анализу;
проверить рамки и ответственность;
выявить критические ограничения;
принять процессное решение.
Gatekeeper не анализирует рынок, не считает экономику и не предлагает решений.

Проверки Gatekeeper:
Определение проблемы
Цель
Ограничения (страна, регион, капитал, сроки)
Ответственность за принятие решений
Проверка на реальность
Законность
Ресурсность (для производственных кейсов)

Возможные исходы:

HARD FAIL — анализ невозможен
RETURN WITH CONDITIONS — требуется уточнение данных
ADMITTED — допуск к анализу
Слой уточнения данных (Clarification Layer)
Clarification Layer — вспомогательный сервис Gatekeeper.
Он активируется только в случае RETURN WITH CONDITIONS и:
запрашивает строго недостающие данные;
не принимает решений;
возвращает управление обратно Gatekeeper.

2. Analysis (многослойный анализ)
Анализ запускается только при статусе ADMITTED.

Назначение:
оценка идеи по нескольким независимым направлениям;
работа исключительно с подтверждёнными данными;
использование внешних источников и специализированных ИИ-модулей.
Примеры аналитических слоёв:
валидация идеи;
рынок и конкуренция;
финансовые и обязательные расходы;
ресурсы и реализуемость.
Аналитические слои не принимают решений самостоятельно и формируют вклад в итоговый вердикт.
Анализ не допускает предположений при нехватке данных.
При отсутствии информации процесс останавливается.

3. Plan (план действий)
План формируется только если результат анализа положительный
или положительный при условиях.

План включает:
этапы реализации;
цели этапов;
контрольные точки;
ключевые риски.
Если итоговый вердикт — «нецелесообразно», план не создаётся.
Принципы проектирования
Проект следует подходу constraint-first design:
сначала ограничения и допуск;
затем анализ;
и только после этого действия.

Система допускает и фиксирует отказ как корректный результат.

Ограничения проекта

Проект не занимается:
мотивацией и коучингом;
управлением персоналом;
принятием решений за пользователя;
генерацией бизнес-идей.
Проект работает строго в рамках закона.

Целевая аудитория

продуктовые и системные архитекторы;
AI / GenAI инженеры (orchestration, decision systems);
аналитические и инвестиционные команды;
предприниматели и стартапы (как прикладной сценарий);
работодатели, оценивающие архитектурное мышление.

Статус проекта

Проект находится в стадии проектирования и формализации архитектуры.
Текущая версия предназначена для демонстрации:
системного мышления;
проектирования сложных AI-контуров;
работы с ограничениями и ответственностью.
Ключевой принцип
Если идея не проходит Gatekeeper — анализ не начинается.
